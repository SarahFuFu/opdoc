租赁系统介绍

该产品是一个基于多租户车辆管理的SAAS系统。尤其可以满足一些车辆体量较小的车企、车辆运营公司等只需要对少量车辆进行管理的客户的需求。不同的公司可以录入自己的车，对自己的车辆信息进行管理、监控。

角色介绍

角色名称 角色介绍
系统管理员
 也称为“超级管理员”。
1.对租户（即租赁公司）进行管理
2.对各个租户的管理员进行管理
3.对租户的角色进行管理
4.查看系统日志
5.管理菜单
租户管理员 即租赁公司管理员。
可以对本公司的角色、机构、本公司的用户进行管理。可以将自己权限范围内的业务信息相关菜单授权给下级角色，但不能对菜单进行增删改。
可以管理自己公司内部的车辆、SIM卡、终端等业务信息。
车辆管理员 租赁公司管理员的下级账号。
只能访问“租户管理员”授权给自己的菜单。菜单上的按钮权限，也需要租户管理员授权后才可见。

租赁平台结构图

  • INTEST可对各个租赁公司进行管理，但无权对租赁公司内部的数据进行管理。各个租赁公司只对本公司自己的数据有访问权限。

租赁平台车辆授权机制

  • 人员组织机构和车辆组织机构是两个独立的部分，不可混淆。
  • 人员组织机构是树形结构，根据角色划分权限。角色是直接和租户关联，由每个租户管理员管理本公司的角色。
  • 每个用户必须至少拥有一个角色，若用户有多个角色，则功能、数据权限取其集合。
  • 车辆组织机构也是树形结构, 每辆车可指定是否为全局车辆（IsGlobal），也可指定树形节点。

系统目标

  1. 需求目标

    • 对不同客户的车辆、人员进行分层级管理
    • 智能查询：用户自定义查询
    • 输出公共平台前后端OP框架，建立统一的开发框架模式

  2. 性能目标

    • 单次请求不超过XX毫秒
    • 能同时支持XX级别用户访问
    • 能抵御各种web攻击

  3. 架构目标

    • 前后端分离：前端VUE 、后台SpringBoot + MyBatis
    • 功能过滤：ApacheShiro
    • 数据过滤：Sql过滤
    • 数据缓存：J2Cache
    • 数据回收站: 增加数据回收数据表
    • 参数管理：用字典对系统级别的参数进行管理
    • 参数配置：业务相关的参数配置信息
